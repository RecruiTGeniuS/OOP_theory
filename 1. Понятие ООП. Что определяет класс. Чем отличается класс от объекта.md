# Понятие ООП. Что определяет класс? Чем отличается класс от объекта?
---
## Билет
---

***Объектно-ориентированное программирование(ООП)*** - парадигма программирования, основанная на представлении программы в виде совокупности объектов, каждый из которых является экземпляром определённого класса, а классы могут образовать иерархию наследования.

>*Парадигма* в программировании - это совокупность идей и понятий, определяющих стиль написания компьютерных программ.

***Класс*** - это тип, определяемый пользователем. Он определяет поведение и характеристики объекта.   
(К примеру, если образно взять класс термос, то его характеристиками будет объём и цвет, а поведением то что он хранит в себе жидкость)

***Чем отличается класс от объекта?***  
Тем что класс - это по сути шаблон, по которому создаются объекты этого класса.  
(Класс: фрукт. Объекты: банан, киви, манго)

>Объект и экземпляр - одно и то же

Выше было онсновное отличие, по сути его должно хватить, но если понадобиться можно ещё дополнить, что класс описывается и определяется единожды, объект же создаётся и создать его можно множетсво раз.

Пример класса:
```c++
class SomeClass
{
public: // Модификатор доступа
    int a; // Поля класса
    int b;
};

int main()
{
    SomeClass obj; // Объект класса SomeClass

    return 0;
}
```
